# Přechod ze serverless portálu na profesionální provoz

## Etapa A — nyní

- GitHub Pages,
- samostatné aplikace,
- vlastní API klíče uživatelů,
- ruční import a export souborů,
- čtecí ukázková knihovna,
- lokální pilotní data.

## Etapa B — po úspěšném pilotu

- školní subdoména,
- malý backend,
- přihlášení školním účtem,
- oddělená databáze,
- knihovna materiálů s oprávněními,
- bezpečná serverová správa API klíče,
- anonymizované souhrnné statistiky.

## Etapa C — plná školní platforma

- role učitel / komise / správce / vedení,
- verzování a schvalování materiálů,
- pracovní postupy mezi aplikacemi,
- auditní logy,
- zálohování a test obnovy,
- testovací a produkční prostředí,
- řízené aktualizace a návrat na předchozí verzi.

## Otázky pro IT

1. Jaký webový backend je škola schopna dlouhodobě provozovat?
2. Je možné použít Docker, Node.js nebo jinou podporovanou technologii?
3. Má škola databázový systém pro oddělenou databázi projektu?
4. Lze použít školní Google nebo Microsoft účty pro přihlášení?
5. Kde budou bezpečně uložena serverová tajemství a API klíče?
6. Jak probíhá zálohování, obnova a monitoring?
7. Kdo nasazuje aktualizace a kdo řeší incidenty?
8. Může být služba dostupná z domova pod školní subdoménou?
